# Contributing
If you are willing to contribute to the project, make sure you have the appropriate tools first.

In case you don't know what's Git or VSCode (IntelliJ, etc.), it's likely better to create an issue, describing what you want to be implemented. Or well, you could learn programming too.

Due to Git being very humorous and not allowing us to properly ignore already pushed files, you'll need to ignore all the configs locally.
